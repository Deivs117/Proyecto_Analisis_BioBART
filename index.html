Sección 3 — El Corazón

Mecanismo de Atención

Cómo el modelo decide qué palabras son relevantes entre sí

Q
Query
La pregunta
¿Qué estoy buscando?

Proviene del token actual. Representa lo que este token quiere saber de los demás.

Q = X · WQ
Ej: "hipertensión" pregunta
¿con qué está relacionada?
K
Key
El índice
¿Qué información tengo?

Proviene de todos los tokens. Actúa como etiquetas en una base de datos.

K = X · WK
Ej: Cada token anuncia
su "tema" o contenido
V
Value
El contenido
¿Cuál es el contenido real?

La información que realmente se transfiere si la atención es alta entre Q y K.

V = X · WV
Ej: El significado semántico
real del token
Fórmula de Scaled Dot-Product Attention
Attention(Q, K, V) = softmax ( QKT / √dk ) · V
QKᵀ = similitud coseno entre tokens √dk = factor de escala (estabilidad de gradientes) softmax = normalización → pesos de atención [0,1]
Multi-Head Attention: Se aplican h = 12 cabezas en paralelo, cada una con sus propias matrices WQ, WK, WV. Los resultados se concatenan → el modelo captura diferentes tipos de relaciones simultáneamente.
Head 1: sintaxis Head 2: correferencia Head 3: semántica ... Head 12

Sección 4

Innovaciones de BioBART

Tareas de pre-entrenamiento y dominio especializado

🔲
Text Infilling
Tarea de corrupción principal
Se enmascaran spans contiguos de texto con un único token [MASK]. El modelo aprende a reconstruirlos.
Original
"The patient has chronic heart failure and comorbidities"
↓ corrupción ↓
Entrada al modelo
"The patient has [MASK] and comorbidities"
Aprende semántica biomédica profunda al reconstruir términos médicos complejos.
🔀
Sentence Permutation
Tarea de coherencia estructural
Las oraciones del documento se mezclan aleatoriamente. El modelo aprende el orden lógico del discurso médico.
Original (orden correcto)
1. Antecedentes del paciente.
2. Diagnóstico principal.
3. Tratamiento prescrito.
↓ permutación ↓
Entrada al modelo (desordenado)
3. Tratamiento prescrito.
1. Antecedentes del paciente.
2. Diagnóstico principal.
Refuerza la coherencia narrativa en reportes clínicos y artículos científicos.
📚
Corpus PubMed
~4.5M abstracts de artículos biomédicos revisados por pares. Dominio altamente especializado.
🔤
Vocabulario BPE Médico
Byte-Pair Encoding adaptado. Reconoce términos como "myocardial", "hypertension" como unidades completas.
🏆
Resultado
SOTA en tareas como diálogo médico, resumen clínico y simplificación de reportes.

Sección 6

Limitaciones

Consideraciones críticas para uso en entornos clínicos

💸
Costo Computacional
~140M parámetros entrenables
Fine-tuning requiere GPUs de alta memoria
Inferencia costosa vs. modelos más pequeños
Pre-entrenamiento: semanas en clúster
Impacto: Barrera de acceso en entornos con recursos limitados
📏
Límite de Contexto
Encoder: máx. 1024 tokens
Decoder: máx. 1024 tokens generados
Historiales clínicos largos quedan truncados
Complejidad cuadrática: O(n²) en atención
Impacto: Inapropiado para notas clínicas extensas sin chunking
⚠️
Alucinaciones Clínicas
Puede generar dosificaciones incorrectas
Síntomas o diagnósticos plausibles pero falsos
Sin mecanismo de verificación factual nativo
El modelo no "sabe" lo que no sabe
⚠ Crítico: Requiere supervisión humana en aplicaciones clínicas reales
Conclusión del Equipo
BioBART representa un avance significativo en NLP biomédico al especializar BART con datos y técnicas de dominio. Sin embargo, su uso en entornos clínicos reales requiere validación rigurosa, supervisión experta y sistemas de verificación factual complementarios.
🧬 Dominio especializado 🔬 Base: PubMed 🏗 Arquitectura: Encoder-Decoder ⚡ Potencial: Alto

Sección 7 — NUEVO

Generación de Texto

Inferencia Autoregresiva de BioBART — Ejemplo Clínico Paso a Paso

🎯
¿Qué es la Inferencia Autoregresiva?

BioBART genera texto un token a la vez. En cada paso t, el decoder recibe los tokens ya generados y consulta mediante Cross-Attention la representación oculta Z producida por el encoder. El token más probable se selecciona y se agrega a la secuencia, convirtiéndose en la entrada del paso t+1 — hasta generar el token <EOS>.

Causal (izq → der) Masked Self-Attention Cross-Attention al Encoder Softmax → Greedy/Beam
0

Decoder: Generación Token a Token

El decoder comienza con <BOS> y genera tokens sucesivos. En cada paso ejecuta dos capas de atención:

① Masked Self-Attention

El token en posición t solo puede ver tokens en posiciones 0…t-1. La máscara causal impide ver el futuro, garantizando la causalidad autoregresiva.

mask[i,j] = -∞ si j > i
② Cross-Attention

La Query Q viene del decoder. Las Keys K y Values V vienen de Z (salida del encoder). El decoder "consulta" todo el contexto de entrada en cada paso.

Q←decoder · K,V←Z_enc
Paso t Token Generado Cross-Attention hacia Z Top Proba
t = 0 <BOS> Señal de inicio de secuencia —
t = 1 "presenta" → "paciente" + [MASK]₁ 0.68
t = 2 "tos" → "fiebre" + "alta" 0.54
t = 3 "seca" → "tos" (paso anterior) + "fiebre" 0.71
t = 4 "disnea" → [MASK]₂ + "tres días" 0.61
t = 5 "asma" → "antecedente" + [MASK]₃ 0.76
t = N <EOS> Señal de fin — detiene la generación —
3

Linear + Softmax → Selección del Token

En cada paso t, el hidden state del decoder [1×768] se proyecta al vocabulario completo mediante una capa lineal y luego normalizada con softmax:
Proyección al vocabulario
P(wt | w<t, Z) = softmax(Wout · ht)
ht ∈ ℝ768 — hidden state del decoder Wout ∈ ℝV×768 — V ≈ 50,265 tokens

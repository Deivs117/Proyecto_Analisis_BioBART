Guía Visual de LLMs: Transformer Encoder-Decoder, BART y BioBART

Material de repaso académico para Procesamiento de Datos Secuenciales (Módulo PLN). Esta guía explica, con foco técnico y aplicado, por qué un modelo generativo biomédico como BioBART es ideal cuando la tarea exige redactar texto nuevo y coherente.

1) Introducción general de la arquitectura

Datos secuenciales

Un texto es una secuencia ordenada de tokens. El significado depende del contexto y de la posición relativa en la frase (ej. negaciones, anáforas, entidades biomédicas).

Paralelización Transformer

A diferencia de RNN/LSTM, el Transformer procesa todos los tokens en paralelo, reduciendo tiempos y mejorando captura de dependencias largas.

Positional Encoding

Como la atención es paralela, se pierde el orden temporal si no se inyecta posición. Se suma una codificación posicional al embedding para preservar la estructura secuencial.

PE(pos,2i)=sin(pos/10000^(2i/d_model))
PE(pos,2i+1)=cos(pos/10000^(2i/d_model))

2) Encoder (el “compresor” del mundo real)

Flujo de entrada y representación contextual

Entrada = Embeddings + Positional Encoding. Cada bloque del encoder aplica Multi-Head Self-Attention bidireccional para que cada token “mire” a toda la secuencia y refine su significado contextual.

Self-Attention multi-cabeza

Por cabeza, se proyectan matrices Q, K, V y se calcula:

Attention(Q,K,V)=softmax((QKᵀ)/√dk)V

Varias cabezas capturan relaciones distintas (sintácticas, semánticas, terminología clínica, etc.).

Add & Norm + Feed Forward

Cada subcapa se estabiliza con conexión residual (Add) y normalización (Norm), seguida de una red Feed Forward (MLP) para transformación no lineal token a token.

Salida del Encoder

El encoder produce estados contextuales finales que funcionarán como memoria fuente del decoder. En cross-attention, estos estados proveen las K y V definitivas.

3) Decoder (el “generador autorregresivo”)

Instante cero y arranque con BOS

La inferencia inicia con <s> / <BOS>. Desde ahí, el decoder predice un token por paso: salidat depende de salida<t + contexto del encoder.

Bloque 1: Masked Self-Attention

Se aplica una máscara triangular superior con -∞ para impedir acceso a posiciones futuras. Así se evita “hacer trampa” durante entrenamiento y se respeta causalidad autoregresiva.

score = (QKᵀ)/√dk + mask, donde maski,j= -∞ si j > i
Bloque 2: Cross-Attention (puente vital)

Q proviene del decoder (lo que va escribiendo). K y V provienen del encoder (fuente original). Conceptualmente: “¿Qué parte de la información de entrada necesito ahora para escribir el siguiente token?”

Bloque 3: Feed Forward + Add & Norm

Después de la integración contextual, el decoder pasa por MLP y normalización residual para producir una representación lista para proyectarse al vocabulario.

4) Etapa final de generación e inferencia

Linear + Softmax

El estado del decoder se proyecta al tamaño del vocabulario. Softmax convierte logits en distribución de probabilidad para seleccionar el siguiente token.

Greedy Search

Elige siempre el token más probable. Es simple y rápido, pero puede caer en repeticiones y bucles semánticos cuando la probabilidad local domina.

Muestreo controlado

Temperature ajusta entropía; Top-K limita candidatos a K tokens; Top-P (nucleus) usa el mínimo conjunto con probabilidad acumulada P. Mejora diversidad manteniendo coherencia.

Criterio de parada: la generación termina al producir <EOS> o alcanzar longitud máxima.

5) Arquitectura BART (base de BioBART)

BART como híbrido Encoder-Decoder

BART combina comprensión contextual global (encoder) con generación autoregresiva (decoder). Es adecuado para resumen, parafraseo, simplificación y generación condicionada.

Contraste con BERT

BERT es solo encoder: excelente para clasificación, extracción y comprensión, pero no está diseñado para redactar texto token a token con un decoder causal.

¿Por qué BioBART en biomedicina?

BioBART adapta BART al dominio biomédico. Esto mejora terminología especializada y permite generar texto clínico/científico coherente, no solo entenderlo. En problemas donde se requiere producción de lenguaje (resúmenes médicos, abstracción de literatura, borradores técnicos), supera enfoques solo-encoder.

Glosario rápido

Q (Query)
Lo que “pregunta” un token para encontrar información relevante.
K (Key)
Claves contra las que se compara la Query para calcular afinidad.
V (Value)
Contenido que se combina ponderadamente según los pesos de atención.
BOS (<BOS> / <s>)
Token de inicio que dispara la generación en el decoder.
EOS (<EOS>)
Token de fin que indica detener la inferencia.
